# Make Something: 토이 프로젝트 아이디어 발굴

소크라테스 문답법으로 사용자의 관심사와 동기를 탐색하고, 5시간 내 완성 가능한 토이 프로젝트 아이디어를 구체화합니다.

## 전제 조건

현재 프로젝트의 테크 스택을 확인합니다:
- `package.json`을 읽어 사용 중인 프레임워크/라이브러리를 파악
- 이 테크 스택을 기반으로 아이디어를 제안해야 합니다

## Step 1: 소크라테스 문답 (관심사/동기 탐색)

`$ARGUMENTS`가 있으면 그것을 시작점으로, 없으면 처음부터 탐색합니다.

### 질문 규칙

- **한 번에 질문 하나.** 절대 두 개 이상 질문하지 마세요.
- `AskUserQuestion`을 사용하고, 가능하면 2-4개 구체적 선택지를 제시하세요.
- 이전 답변을 기반으로 다음 질문을 만드세요.
- 사용자가 스스로 자신의 필요를 발견하도록 유도하세요. 답을 주지 말고 질문으로 이끄세요.

### 질문 차원

다음 차원을 순서대로 탐색합니다. 모든 차원을 다 물어볼 필요는 없습니다.

1. **일상의 불편함** — 평소 반복적으로 하는 귀찮은 작업이 있는가?
2. **관심 분야** — 요즘 빠져있거나 관심 가는 주제가 있는가?
3. **학습 동기** — 배워보고 싶은 기술이나 개념이 있는가?
4. **대상 사용자** — 본인만 쓸 건지, 다른 사람에게도 보여줄 건지?
5. **복잡도 선호** — 단순하고 완성도 높은 것 vs 도전적인 것?

### 종료 조건

다음 중 하나면 질문을 멈추고 Step 2로 이동합니다:

- 사용자가 "됐어", "충분해", "이 정도면 돼" 같은 신호를 보냄
- 관심 분야 + 만들고 싶은 방향이 충분히 구체화됨
- 3회 이상 질문한 후 새로운 발견이 없음

## Step 2: 아이디어 후보 제시

문답 결과를 종합하여 **2-3개의 아이디어 후보**를 제시합니다.

### 후보 제시 규칙

- 각 아이디어는 다음을 포함합니다:
  - **한 줄 설명**: 무엇을 만드는지 명확하게
  - **핵심 기능 3-5개**: 5시간 내 구현 가능한 범위로 제한
  - **왜 이 아이디어인지**: 문답에서 나온 관심사/동기와의 연결점
- `AskUserQuestion`으로 후보를 제시하고 사용자가 선택하게 합니다
- 사용자가 수정하고 싶다면 반영하여 아이디어를 조정합니다

### 스코프 가이드라인

5시간 내 완성 가능한 규모로 제한합니다:
- 페이지 1-3개
- 핵심 기능 3-5개
- 외부 서비스 연동은 1-2개까지
- 인증/결제 등 복잡한 기능은 제외

## Step 3: 웹 리서치

확정된 아이디어를 기반으로 `WebSearch`를 사용하여 관련 무료 도구를 조사합니다.

### 리서치 범위

다음 카테고리별로 검색합니다:

1. **무료 API/서비스** — 아이디어에 필요한 데이터나 기능을 제공하는 무료 API (예: 공공 데이터 API, 무료 티어 BaaS)
2. **npm 패키지** — 핵심 기능 구현에 도움되는 오픈소스 라이브러리
3. **디자인/UI 리소스** — 무료 아이콘, 이미지, 폰트 등
4. **관련 스킬 검색** — `npx skills find [아이디어 키워드]`로 구현에 도움될 스킬을 검색합니다.
   - 검색 결과에서 유용한 스킬이 있으면 사용자에게 소개
   - 설치 여부를 확인한 후 `npx skills add <package> -g -y`로 설치

### 리서치 규칙

- 각 카테고리에서 최소 2-3개 옵션을 찾습니다
- 무료 사용 가능 여부를 반드시 확인합니다
- 현재 테크 스택(Next.js, React, TypeScript)과 호환되는지 확인합니다
- `references/free-tools.md`를 참고하여 이미 알려진 무료 도구도 포함합니다

## Step 4: 아이디어 정리 문서 생성

리서치 결과를 포함한 아이디어 정리 문서를 작성합니다.

### 문서 포맷

`references/idea-template.md`를 읽고 그 형식에 맞춰 작성하세요.

### 작성 규칙

- 기능 목록은 우선순위 순으로 정렬합니다
- 각 기능에 대해 관련 무료 도구를 매핑합니다
- 도구 추천에는 무료 사용 조건(한도, 제약)을 명시합니다
- 현재 테크 스택 정보를 포함합니다

## Step 5: 저장

최종 문서를 `artifacts/` 디렉토리에 저장합니다.

- 파일명: `artifacts/<project-name>/idea.md`
- 저장 전 사용자에게 파일명 확인
- 저장 후 다음 단계 안내: "요구사항을 구체화하려면 `/writing-spec`을 실행하세요."
