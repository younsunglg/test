## Overview
매일 아침 Bloomberg, Reuters, CNBC 등 금융 뉴스를 한 화면에서 모아보고,
AI 감성 스코어(Bullish/Bearish/Neutral)와 Reddit 투자 커뮤니티 반응을 함께 확인해
주식 투자 판단 자료를 빠르게 수집·정리하는 개인 투자 리서치 대시보드.

## 시나리오

### 1. 앱 최초 진입 — 기본 소스 피드 표시
[상황] 사용자가 앱에 처음 진입하면
[동작] Reuters와 CNBC Markets RSS가 기본 등록된 상태로 최신 기사 목록을 표시한다

성공 기준:
- [ ] 소스 탭에 "All", "Reuters", "CNBC" 탭이 표시된다
- [ ] "All" 탭 선택 시 두 소스의 기사가 합쳐져 발행 시각 내림차순으로 표시된다
- [ ] 각 기사 카드에 소스명(예: "Reuters"), 제목, 발행 시각("2시간 전")이 표시된다
- [ ] Bullish / Bearish / Neutral 배지가 각 카드에 표시된다

### 2. 앱 최초 진입 — 피드 로딩 중 상태
[상황] 앱에 처음 진입했고 RSS API 응답이 아직 도착하지 않은 상태에서
[동작] 피드를 불러오는 동안 로딩 상태를 표시한다

성공 기준:
- [ ] 기사 목록 영역에 로딩 스피너가 표시된다
- [ ] 로딩 중에는 기사 카드가 0개이다

### 3. RSS 소스 추가
[상황] 소스 목록에 "FinancialJuice"가 없는 상태에서
[동작] 사용자가 소스 추가 입력란에 RSS URL을 입력하고 추가 버튼을 클릭하면 소스가 등록되고 피드를 불러온다

성공 기준:
- [ ] 유효한 URL(`https://feeds.financialjuice.com/...`) 입력 후 추가 → 소스 탭에 "FinancialJuice" 탭이 생긴다
- [ ] 새 소스의 기사가 "All" 탭에 합쳐져 표시된다
- [ ] 빈 URL 입력 후 추가 → "URL을 입력해주세요" 오류 메시지가 표시된다
- [ ] 중복 URL 입력 후 추가 → "이미 등록된 소스입니다" 오류 메시지가 표시된다
- [ ] URL 형식이 아닌 텍스트(`not-a-url`) 입력 후 추가 → "올바른 URL 형식이 아닙니다" 오류 메시지가 표시된다

### 4. 소스별 탭 필터
[상황] "All", "Reuters", "CNBC" 탭이 있고 전체 기사 목록이 표시된 상태에서
[동작] 사용자가 "Reuters" 탭을 클릭하면 Reuters 기사만 표시된다


성공 기준:
- [ ] "Reuters" 탭 클릭 → 모든 카드의 소스명이 "Reuters"로 표시된다
- [ ] "All" 탭 클릭 → 모든 소스 기사가 다시 합쳐져 표시된다
- [ ] 탭 전환 시 스크롤 위치가 상단으로 초기화된다

### 4. 키워드 검색
[상황] 여러 기사가 표시된 뉴스 피드에서
[동작] 사용자가 검색창에 키워드를 입력하면 관련 기사만 필터링된다

성공 기준:
- [ ] "Fed" 입력 → 제목에 "Fed" 포함된 기사만 표시된다
- [ ] 검색 결과가 없으면 "검색 결과가 없습니다" 문구가 표시된다
- [ ] 검색창을 비우면 전체 기사 목록으로 돌아온다

### 5. 북마크 및 메모 추가
[상황] 기사 카드가 표시된 상태에서
[동작] 사용자가 기사 카드의 북마크 아이콘을 클릭하면 북마크가 저장되고, 메모 아이콘 클릭 시 메모를 입력할 수 있다

성공 기준:
- [ ] 북마크 아이콘 클릭 → 아이콘이 채워진 상태(★)로 바뀌고 페이지 새로고침 후에도 유지된다
- [ ] 메모 아이콘 클릭 → 메모 입력 다이얼로그가 열린다
- [ ] "AAPL 실적 발표 주목" 메모 저장 → 카드에 메모 미리보기 텍스트가 표시된다
- [ ] "북마크" 탭 클릭 → 북마크된 기사만 표시된다

### 6. 수동 새로고침
[상황] 뉴스 피드가 표시된 상태에서
[동작] 사용자가 새로고침 버튼을 클릭하면 모든 소스를 다시 페치한다

성공 기준:
- [ ] 새로고침 버튼 클릭 → 버튼에 로딩 스피너가 표시된다
- [ ] 페치 완료 → "방금 전 업데이트" 텍스트가 표시된다
- [ ] 새로 불러온 기사가 목록 상단에 추가된다

### 7. Reddit 커뮤니티 반응 섹션
[상황] 대시보드 하단 Reddit 섹션이 표시된 상태에서
[동작] ApeWisdom/Tradestie API에서 가져온 종목 언급 순위와 r/investing 최신 게시글을 표시한다

성공 기준:
- [ ] WSB Top 종목 리스트(순위, 티커, 언급 수)가 표시된다 (예: "1. AAPL 342회")
- [ ] r/investing 최신 게시글 제목 3개 이상이 표시된다
- [ ] 각 Reddit 게시글 클릭 → 해당 링크가 새 탭으로 열린다

### 8. Reddit 섹션 — API 오류 상태
[상황] 대시보드 하단 Reddit 섹션이 로드 중인 상태에서
[동작] ApeWisdom 또는 Reddit API 요청이 실패하면 오류 상태를 표시한다

성공 기준:
- [ ] API 실패 시 "Reddit 데이터를 불러올 수 없습니다" 오류 문구가 표시된다
- [ ] 재시도 버튼이 표시된다
- [ ] 재시도 버튼 클릭 → API를 다시 호출한다

## 범위

### 포함
- RSS 소스 등록/삭제 및 피드 표시
- Alpha Vantage 기반 Bullish/Bearish/Neutral 감성 배지
- 소스별 탭 필터 + 키워드 검색(fuse.js)
- 북마크 + 투자 메모 (localStorage 영속)
- Reddit WSB Top 종목 + r/investing 최신글 (하단 섹션)
- 수동 새로고침

### 제외
- 자동 주기적 갱신 (Alpha Vantage 한도 초과 위험)
- X(Twitter) 실시간 연동 (X API 유료화로 현실적으로 불가)
- 로그인/회원가입 (개인 도구이므로 불필요)
- 가격 차트, 포트폴리오 추적 (이번 스코프 초과)
- Bloomberg 공식 RSS (2019년 이후 지원 중단)

## 전제 조건
- Alpha Vantage API 키를 환경변수(`ALPHA_VANTAGE_API_KEY`)에 등록한다
- RSS 외부 호출은 Next.js Route Handler에서 서버사이드로 처리한다 (CORS 우회)
- 북마크·메모·소스 목록은 localStorage에 저장한다 (서버 불필요)
- Reuters, CNBC RSS는 앱 번들에 기본값으로 하드코딩되어 있다

## 미결정 사항
- 소스 삭제 시 해당 소스의 북마크 기사도 함께 삭제할지, 유지할지
- Alpha Vantage 일일 한도(500 req) 초과 시 기사 감성 배지를 숨길지, 캐시된 값을 유지할지
